1. 객체 리터럴

1. 객체란?

  • 자바스크립트는 객체기반의 프로그래밍 언어.
  • 원시 값을 제외한 나머지 값들 (함수, 배열, 정규 표현식) 은 모두 객체.
  • 원시타입 > 단 하나의 값 (변경 불가능한 값)
  • 객체타입 > 다양한 타입의 값들을 하나의 단위로 구성한 복합적인 자료구조 (변경 가능한 값)
  • 객체 : 키, 값으로 구성된 프로퍼티의 집합.
  • 모든 값은 프로퍼티 값 가능.
  • 함수도 일급객체. 값으로 취급 가능. = 메소드 Method
  • 프로퍼티 > 상태, 메소드 > 동작으로 객체 구조화 할 수 있어 유용하다.

2. 객체 리터럴에 의한 객체 생성

  • 자바스크립트는 프로토타입 기반 객체지향 언어로, 다양한 객체 생성 방법이 존재한다.
  • 객체 리터럴, Object 생성자 함수, 생성자 함수, Object.create 메소드, 클래스(ES6)
  • 객체 리터럴 : 객체를 생성하는 표기법. 값으로 평가되는 표현식. {} 내에 프로퍼티를 정의. 코드블록이 아니므로 ; 세미콜론 붙여야 함.

3. 프로퍼티

  • 객체는 프로퍼티들의 집합.
  • 키, 값으로 구성하고 쉼표로 구분.
  • 프로퍼티 키 > 모든 문자열 or Symbol 값. 식별자 역할. 식별자 네이밍 규칙을 따르지 않은 이름에는 ‘’ 사용. 문자열이나 Symbol값 이외의 값을 사용하면 암묵적 타입 변환에 의해 문자열로 변환됨.
  • 프로퍼티 값 > 자바스크립트에서 사용가능한 모든 값.
  • 중복 선언하면 나중에 선언한 키를 덮어씀.

4. 메소드

  • 자바스크립트에서 사용할 수 있는 모든 값 > 프로퍼티 값 사용 가능.
  • 함수 > 값으로 취급 가능.
  • 그러므로, 함수 > 프로퍼티 값으로 사용가능.
  • 객체에 있는 함수는 메소드라 부른다.

5. 프로퍼티 접근

  • 마침표 표기법: 객체표현식.프로퍼티키 (ex/ person.name)
  • 대괄호 표기법: 객체표현식[‘프로퍼티키’] (ex/ person[‘name’])
  • [‘’] 따옴표로 감싸지 않으면 식별자로 해석 > Reference Error
  • 객체에 존재하지 않는 프로퍼티에 접근하면 > undefined

6. 프로퍼티 값 갱신

  • 이미 존재하는 프로퍼티에 값을 할당하면 프로퍼티 값이 갱신됨.
  • ##3 에서 언급했던 중복 선언 이야기인 듯.

7. 프로퍼티 동적 생성

  • 존재하지 않는 프로퍼티에 값을 할당하면 동적으로 생성, 추가되고 프로퍼티 값이 할당됨.

8. 프로퍼티 삭제

  • delete 연산자는 객체의 프로퍼티를 삭제한다.

9. ES6에서 추가된 객체 리터럴의 확장 기능

A. 프로퍼티 축약 표현

  • 변수 이름과 프로퍼티 키가 동일한 이름일 때, 프로퍼티 키 생략가능. 프로퍼티 키는 변수 이름으로 자동생성됨.

B. 프로퍼티 키 동적 생성

  • 프로퍼티 키 동적 생성 (계산된 프로퍼티 이름) : 프로퍼티 키로 사용할 표현식을 [] 대괄호로 묶는다.

C. 메소드 축약 표현

  • 프로퍼티 값으로 함수를 할당해 메소드를 정의.
  • ES6 : function 키워드를 생략해 축약표현. 프로퍼티에 할당한 함수와 다르게 동작.

1. 타입 변환과 단축 평가

1. 타입 변환

  • 명시적 타입 변환, 타입 캐스팅 : 개발자의 의도로 값의 타입을 변환하는 것.

2. 암묵적 타입 변환

  • 개발자의 의도와 관계없이 데이터 타입이 강제 변환 되는 것.
  • 명시적 타입 변환이나 암묵적 타입 변환이 기존 원시값(=변경 불가능 한 값)을 직접 변경하는 것은 아니고,
    다른 타입의 원시값을 새로 생성함.

A. 문자열 타입으로 변환

  • +연산자의 피연산자 중 하나 이상이 문자열이면 문자열 연결 연산자로 동작. 문자열이 아닌 피연산자를 문자열로 암묵적 타입 변환.
  • 피연산자 + ‘’ > 문자열로 타입변환.
  • symbol 타입의 경우만 TypeError.

B. 숫자 타입으로 변환

  • 산술연산자의 피연산자 중에서 숫자타입이 아닌 피연산자를 숫자타입으로 암묵적 타입 변환.
  • 숫자타입으로 변환이 안 될 경우에는 NaN 반환.
  • 비교연산자는 숫자 타입으로 암묵적 타입 변환 후 불리언 값으로 반환.
  • +’string’, +undefined, +{}, +[10, 20], +(function(){}) > NaN
  • +Symbol() > TypeError.

C. 불리언 타입으로 변환

  • 조건식의 평가 결과를 불리언 타입으로 암묵적 타입 변환.
  • 불리언 타입이 아닌 값을 truthy 값(true), falsy 값(false)으로 구분하고 타입 변환.
  • false로 평가되는 falsy 값.
    : false, undefined, null, 0, -0, NaN, ‘’(빈문자열)
    이 외는 모두 true로 평가되는 truthy 값.

3. 명시적 타입 변환

A. 문자열 타입으로 변환

  • string 생성자 함수를 new 연산자 없이 호출하는 방법.
  • Object.prototype.toString 메소드를 사용.
  • 문자열 연결 연산자를 이용.
    (암묵적 타입 변환 방법을 이용해서 개발자의 의도로 사용.)

B. 숫자 타입으로 변환

  • Number 생성자 함수를 new 연산자 없이 호출하는 방법.
  • parseInt, parseFloat 함수를 사용하는 방법.
    (문자열만 변환가능, ‘숫자’가 들어있는 문자열만 해당.)
  • +단항 산술 연산자를 이용하는 방법.
  • *산술 연산자를 이용하는 방법.

C. 불리언 타입으로 변환

  • Boolean 생성자 함수를 new 연산자 없이 호출하는 방법
  • !부정 논리 연산자를 두 번 사용하는 방법

4. 단축평가

  • &&, || 는 논리연산의 결과를 타입 변환하지 않고, 피연산자를 그대로 반환한다.

&& AND

  • 두 개의 피연산자가 모두 true 일 때 true.
  • 왼쪽 > 오른쪽 평가 진행.
  • ‘Cat’ && ‘Dog’ 경우 ‘’(빈 문자열) 외에 문자열은 truthy 값이고, ‘Cat’ 평가 후 ‘Dog’까지 평가해 truthy 이므로 마지막에 평가한 ‘Dog’를 반환함.

|| OR

  • 두 개의 피연산자 중 하나만 true 여도 true.

  • 왼쪽 > 오른쪽 평가 진행.

  • ‘Cat’ || ‘Dog’ 경우 ‘Cat’ 만 평가해도 이미 true 이므로 바로 ‘Cat’ 을 반환.

  • 단축평가를 사용하면 if문 대체 가능.

  • 주어진 조건이 truthy 값 일 때 &&

  • 주어진 조건이 false 값 일 때 ||

자바스크립트 복습 - 리터럴, 표현식과 문, 데이터 타입

1. 리터럴

사람이 이해할 수 있는 문자나 기호를 이용해 값을 생성하는 표기 방식.
컴퓨터 자체의 언어로 코드를 사용하면 비효율적이니 사람들끼리 미리 약속해서 값을 생성할때 이런 표기법을 쓰자고 만든 것.
숫자 정수 문자열 불리언 null undefined 객체 배열 함수 정규표현식 리터럴

2. 표현식인 문과 표현식이 아닌 문

값으로 평가될 수 있는 문, 표현식이 평가되면 값을 생성하거나 참조.
그래서 1+2=3 이 동치인 것처럼 표현식=값이므로 값이 들어가는 자리에 표현식을 모두 넣을 수 있고,
표현식인 문과 아닌 문을 구별할 때 변수에 할당해보면 값이 들어가는 자리에 표현식이 들어가는 것이므로 오류가 없다면 표현식이 맞다.

자바스크립트 복습 - 변수

1. 변수

하나의 값을 저장하기 위해 확보한 메모리 공간 또는 그 공간의 메모리주소로 연결하기 위해 붙인 이름.
변수가 필요한 이유 - 연산하거나 등. 저장해 둔 값을 재사용하기 위해서.
변수에 값을 저장하는 것을 할당, 값을 읽어들이는 것을 참조.

2. 식별자

어떤 값을 구분하기 위해 붙인 고유한 이름.
값이 아니라 메모리주소를 기억하고 있음.
변수에 붙인 이름을 말하는데 클래스나 함수이름도 해당된다.

3. 변수 선언

var let const 키워드를 사용해 값을 저장할 공간을 확보할 준비를 하는 것
변수를 선언하면 undefined 라는 값을 암묵적으로 할당

4. 변수 호이스팅

변수 선언은 코드 위치에 관계없이 런타임 실행 이전에 미리 읽어들이는 자바스크립트 만의 특징.
코드 위치를 강제로 바꾸는 것은 아니고 실행만 먼저 한다.

5. 값의 할당

할당 연산자 = 를 사용하여 우변의 값을 좌변의 변수에 할당.
undefined는 그대로 유지하고 새로운 메모리 공간을 확보하고 할당값을 저장한다.

6. 값의 재할당

이미 값이 할당되어 있는 변수에 새로운 값을 또 다시 할당하는 것.
재할당이 안된다면 상수 const

7. 식별자 네이밍 규칙

카멜케이스
스네이크케이스
파스칼케이스
헝가리안케이스

javascript - control flow statement

Posted on 2020-02-22 In Learn web development , javascript

1. 제어문

  • 주어진 조건에 따라 코드 블록을 실행하거나 반복 실행 할 때 사용.

  • 실행 흐름을 인위적으로 제어 가능하지만, 가독성을 해치는 단점.

  • forEach, map, filter, reduce 와 같은 고차 함수를 사용하여 복잡성을 해결.

    1. 블록문

    • 문을 중괄호로 묶은 것.
    • 코드 블록, 블록.
    • 하나의 실행 단위.
    • 제어문이나 함수를 정의할 때 사용.
    • 블록문의 끝에는 세미콜론을 붙이지 않음.

    2. 조건문

    • 조건식의 평가 결과에 따라 블록문의 실행을 결정.
    • 조건식은 불리언 값으로 평가될 수 있는 표현식.

    A. if…else 문

    • ~ 하다면 ~ 하고, 그렇지 않다면 ~ 해라.

    • 주어진 조건식의 평가 결과, 논리적 참/거짓 에 따라 실행할 블록문을 결정.

    • 참(true)일 경우, if 문 다음의 코드블록이 실행.

    • 거짓(false)일 경우, else 문 다음의 코드블록이 실행.

    • 조건식은 불리언 값으로 평가되어야 함. 만약 불리언 값이 아니라면 암묵적으로 데이터 타입이 불리언 값으로 강제변환되어 실행할 코드 블록을 결정.

    • 실행 될 코드 블록을 늘리고 싶으면 else if 문을 사용.

    • if 문만 사용 가능.

    • if 문과 else 문은 한 번씩만 사용 가능하지만, else if 문은 여러 번 사용 가능.

    • 코드 블록 내의 문이 하나라면 중괄호 생략 가능.

    • 대부분의 if…else 문은 삼항 조건 연산자로 바꿀 수 있음. 삼항 조건 연산자는 값처럼 사용할 수 있기 때문에 변수에 할당 가능하지만 if…else 문은 변수에 할당할 수 없다.

    B. switch 문

    • 주어진 표현식을 평가하여 그 값과 일치하는 표현식을 갖는 case 문으로 실행 순서를 이동시킴.

    • 일치하는 표현식을 갖는 case 문이 없다면 실행 순서는 default 문으로 이동.

    • if…else 문은 참/거짓으로 실행할 코드 블록을 결정, switch 문은 다양한 상황(case)에 따라 실행할 코드 블록을 결정.

    • break 를 사용하지 않으면 모든 문이 실행되어 마지막 문까지 재할당 됨. > fall through 폴스루.

    • fall through 를 활용하면 여러개의 case 를 하나의 조건으로도 사용가능.

    3. 반복문 (Loop Statement)

    • 주어진 조건식의 평가 결과가 참인 경우 코드 블록을 실행.
    • 그 후, 조건식을 다시 검사하여 여전히 참인 경우 코드 블록을 다시 실행. 조건식이 거짓이 될 때까지 반복.

    A. for 문

    • 조건식이 거짓으로 판별될 때까지 코드 블록을 반복 실행.
    • 변수 선언문, 조건식, 증감식은 모두 옵션이므로 반드시 사용할 필요는 없다.
    • 어떤 식도 선언하지 않으면 무한루프.
    • for 문 내에 for 문을 중첩해 사용할 수 있음.

    B. while 문

    • 주어진 조건식의 평가 결과가 참이면 코드 블록을 계속 반복 실행. 거짓이 되면 실행 종료.
    • 조건식의 평가 결과가 불리언 값이 아니면 불리언 값으로 강제변환되어 참/거짓 구별.
    • 조건식의 평가 결과가 언제나 참이면 무한루프.
    • 무한루프 탈출 : 코드 블록 내에 if 문으로 탈출 조건을 만들고 break 문으로 코드 블록 탈출.

    C. do…while 문

    • 코드 블록을 먼저 실행하고 조건식을 평가.
    • 코드 블록은 무조건 한 번 이상 실행 된다.

    D. break 문

    • 코드 블록을 탈출.
    • 레이블 문(식별자가 붙은 문), 반복문, switch 문의 코드 반복을 탈출.
    • 이 외에 사용하면 SyntaxError 문법에러 발생.
    • 레이블 문은 프로그램의 실행 순서를 제어하기 위해 사용. 레이블 문을 탈출하려면 break 문에 레이블 식별자를 지정.
    • 중첩된 for 문의 내부 for 문에서 break 문을 실행하면 내부 for 문을 탈출하여 외부 for 문으로 진입한다. 이 때 내부 for 문이 아닌 외부 for 문을 탈출하려면 레이블 문을 사용한다.
    • 이 외의 경우 레이블 문은 권장하지 않음. 흐름이 복잡해지고 가독성이 나빠져서 오류 발생 확률 높음.
    • break 문은 반복문, switch 문에서도 사용가능. 이 경우에는 레이블 식별자를 지정하지 않음.

    E. continue 문

    • 반복문의 코드 블록 실행을 현 지점에서 중단하고 반복문의 증감식으로 이동.
    • break 문처럼 반복문을 탈출하지는 않는다.
    • if 문 내에서 실행해야 할 코드가 한 줄이라면 continue 사용하지 않는 것이 가독성면에서 좋으나, 코드가 길다면 continue 문을 사용하는 것이 더 좋다.

javascript - operator

Posted on 2020-02-22 In Learn web development , javascript

1. 연산자

  • 하나 이상의 표현식을 대상.

  • 산술, 할당, 비교, 논리, 타입, 지수연산(operation)등을 수행해 하나의 값을 만든다.

  • 연산의 대상 > 피연산자 : 값으로 평가될 수 있는 표현식.

  • 피연산자, 연산자의 조합도 값으로 평가될 수 있는 표현식.

  • 연산자는 값으로 평가된 피연산자를 연산해 새로운 값을 만듦.

    1. 산술 연산자 (Arithmetic Operator)

    • 피연산자를 대상으로 수학적 계산을 수행해 새로운 숫자 값을 만듦.
    • 산술 연산이 불가능한 경우, NaN을 반환.

    A. 이항 산술 연산자 (Binary Arithmetic Operator)

    • 2개의 피연산자를 산술 연산하여 숫자 타입의 값을 만듦.
    • 피연산자의 값을 변경하는 부수 효과(Side effect)가 없다.
    • 어떤 산술 연산을 해도 피연산자의 값이 바뀌는 경우는 없다. 언제나 새로운 값을 만듦.

    B. 단항 산술 연산자 (Unary Arithmetic Operator)

    • 1개의 피연산자를 산술 연산하여 숫자 타입의 값을 만듦.
    • 피연산자의 값을 변경하는 부수효과가 있다.
    • 증가++/감소– 연산을 하면 피연산자의 값을 변경하는 암묵적 할당이 이루어짐.

    • ++/– 위치

      • 전위 증가/감소 연산자: 피연산자 앞에 위치 > 먼저 피연산자의 값을 증가/감소시킨 후, 다른 연산을 수행.

      • 후위 증가/감소 연산자: 피연산자 뒤에 위치 > 먼저 다른 연산을 수행한 후, 피연산자의 값을 증가/감소 시킴.

      • +단항 연산자는 피연산자에 어떠한 효과도 없고, 음수를 양수로 반전하지도 않음.

      • 숫자 타입이 아닌 피연산자에 사용하면 숫자타입으로 변환하여 반환.

      • 피연산자를 변경하는 것은 아님.

      • 숫자 타입으로 변환한 값을 생성해서 반환. 부수효과는 없음.

      • -단항 연산자는 피연산자의 부호를 반전한 값을 반환.

      • 숫자 타입이 아닌 피연산자를 사용하면 부호를 반전한 값을 생성해서 반환. 부수효과는 없음.

    C. 문자열 연결 연산자

    • +연산자는 피연산자 중 하나 이상이 문자열인 경우, 문자열 연결 연산자로 동작.
    • 그 외의 경우는 덧셈연산자로 동작.
    • 암묵적 타입 변환(Implicit coercion), 타입 강제 변환(Type coercion) : 개발자의 의도와는 관계없이 ‘1 + true’ 를 연산하면 true를 숫자타입인 1로 강제변환 한 후 1을 더해 2라는 값을 반환한다.

    2. 할당 연산자 (= 대입연산자)

    • (=)수학에서는 같다는 의미 > 프로그래밍에서는 할당연산자.
    • 우항에 있는 피연산자의 평가 결과를 좌항에 있는 변수에 할당.
    • 좌항의 변수에 값을 할당하므로 변수의 값이 변하는 부수 효과가 있음.
    • 할당문은 값으로 평가되는 표현식인 문.
    • 할당문을 다른 변수에 할당할 수도 있다. 활용해 여러 변수에 동일한 값을 연쇄 할당 가능.

    3. 비교 연산자 (Comparison Operator)

    • 좌항과 우항의 피연산자를 비교, 결과 불리언 값 반환.
    • if문, for문 제어문의 조건식에서 주로 사용.

    A. 동등 / 일치 비교 연산자

    • ==, != 동등, 부동등 비교
      • 암묵적 타입 변환을 통해 타입을 일치시킨 후, 같은 값인지 비교. 타입까지 같은지 비교할 수 없음.
    • ===, !== 일치 비교, 불일치 비교
      • 타입도 같고, 값도 같은 경우에 한해 true 반환.
    • NaN 주의: 자신과 일치하지 않는 유일한 값. NaN === NaN 비교시, false 반환. 숫자가 NaN인지 조사하려면 빌트인 함수 isNaN 사용해야함.
    • 숫자 0 주의: 0 === -0, 0 == -0 모두 true.
    • Object.is 메소드: 예측 가능한 정확한 비교 결과를 반환.

    B. 대소 관계 비교 연산자

    • 피연산자의 크기를 비교하여 불리언 값을 반환.

    4. 삼항 조건 연산자 (Ternary Operator)

    • 조건식의 평가 결과에 따라 반환할 값을 결정.
    • 자바스크립트의 유일한 삼항 연산자. 부수 효과 없음.

    조건식 ? 조건식이 true 일 때 반환할 값 : 조건식이 false 일 때 반환할 값

    • 조건식: 불리언 타입의 값으로 평가될 표현식.
    • 평가 결과가 불리언 값이 아니면 불리언 값으로 암묵적 타입 변환 됨.
    • if…else 문을 사용해도 동일한 처리가 가능하지만,
      if…else 문은 표현식이 아닌 문이므로 값처럼 사용할 수 없다.
    • 삼항 조건 연산자 표현식은 값처럼 다른 표현식의 일부가 될 수 있어 유용함.
    • 수행해야 할 문이 하나라면 삼항 조건 연산자 표현식 사용, 여러 개라면 if…else 문 사용.

    5. 논리 연산자 (Logical Operator)

    • 우항과 좌항의 피연산자를 논리 연산한다.
    ! = NOT
    : 반전
    && = AND
    : 둘 다 true 일 때, true. 하나라도 false 일 땐 false.
    || = OR
    하나라도 true 라면, 결과물은 true. 둘 다 false 일 때만 false.
    • 연산 순서

    NOT > AND > OR
    ! > && > ||

    • ! : 언제나 불리언 값 반환. 피연산자가 불리언 값일 필요는 없음. 암묵적 타입 변환해서 반환.
    • &&, || : 평가 결과는 불리언 값이 아닐 수도 있음. 언제나 2개의 피연산자 중 어느 한쪽으로 평가된다.
    • 드 모르간의 법칙 : 논리합은 논리곱과 부정기호로, 논리곱은 논리합과 부정기호로 표현할 수 있음.
      not (A or B) = (notA) and (notB)
      not (A and B) = (notA) or (notB)

    6. 쉼표 연산자

    • 왼쪽 피연산자부터 차례대로 피연산자를 평가하고 마지막 피연산자의 평가가 끝나면 마지막 피연산자의 평가 결과를 반환.

    7. 그룹 연산자

    • 피연산자인 표현식을 가장 먼저 평가. 연산자의 우선 순위를 조절할 수 있다.

    8. typeof 연산자

    • 피연산자의 데이터 타입을 문자열로 반환.
    • string, number, boolean, undefined, symbol, object, function 중 하나를 반환.
    • null을 반환하는 경우는 없다. object 를 반환. (자바스크립트의 버그) null 타입을 확인할 땐 typeof 연산자가 아니라 일치 연산자 (===) 를 사용.
    • 선언하지 않은 식별자를 typeof 연산자로 연산해 보면 ReferenceError 가 아니라 undefined 를 반환.

    9. 지수 연산자

    • ES7 에서 새롭게 도입.
    • 좌항의 피연산자를 밑으로, 우항의 피연산자를 지수로 거듭 제곱하여 숫자 타입의 값을 반환.
    • 이전에는 Math.pow 메소드 사용.
    • 음수를 계산하려면 괄호로 묶어야 한다.
    • 모든 이항 연산자보다 우선 순위가 높다.

    10. 연산자의 부수 효과

    • 할당(=) 연산자: 변수 값이 변하는 부수 효과.
    • 증가(++)/감소(–)연산자: 피연산자의 값을 변경하는 부수 효과.
    • delete 연산자: 객체의 프로퍼티를 삭제하는 부수 효과.

javascript - data type

Posted on 2020-02-20 Edited on 2020-02-22 In Learn web development , javascript

1. 데이터 타입 (Data type)

  • 값의 종류.
  • 자바스크립트의 모든 값은 데이터 타입을 갖는다.
  • ES6는 7개의 데이터 타입 제공.
  • 원시 타입 (primitive type)
    • 숫자 타입: number
    • 문자열 타입: string
    • 불리언 타입: boolean
    • undefined 타입
    • null 타입
    • symbol 타입
  • 객체 타입 (object/reference type)
    • 객체, 함수, 배열 등.

1. 숫자 타입 (number)

  • 정수와 실수 구분없이 하나의 숫자 타입만 존재.
  • 모든 수를 실수로 처리.
  • 추가적으로 3가지 특별한 값들도 표현 가능.
    • Infinity : 양의 무한대
    • -Infinity : 음의 무한대
    • NaN : 산술 연산 불가 (not-a-number)

2. 문자열 타입 (string)

  • 텍스트 데이터를 나타내는데 사용.
  • 0개 이상의 UTF-16 16bit 유니코드 문자들의 집합으로 전세계 대부분의 문자를 표현할 수 있다.
  • ‘’, “”, ``으로 텍스트를 감싼다.
  • 문자열은 원시타입이며 변경 불가능한 값 (immutable value). 문자열이 생성되면 그 문자열을 변경할 수 없다는 것을 의미.

3. 템플릿 리터럴 (template literal)

  • ES6부터 도입.

  • 런타임에 일반 문자열로 변환되어 처리.

  • ``(backtick) 백틱 사용.

    3-1. 멀티라인 문자열(Multi-line string)

    • 일반 문자열 내에서 개행을 하려면 이스케이프 시퀀스(Escape sequence)를 사용하여야 함.
    • 템플릿 리터럴 내에서는 이스케이프 시퀀스 사용 없이도 개행이 허용되며 모든 공백도 적용됨.

    3-2. 표현식 삽입(Expression interpolation)

    • 문자열은 문자열 연산자 +를 사용해 연결할 수 있다.
    • +연산자는 피연산자 중 하나 이상이 문자열인 경우, 문자열 연산자로 동작. 그 외의 경우는 덧셈 연산자로 동작.
    • 템플릿 리터럴 내에서는 표현식 삽입을 통해 문자열 연산자보다 가독성 좋고 간편하게 문자열을 조합할 수 있다.
    • ${ }으로 표현식을 감싼다. 이 때 표현식의 평가 결과가 문자열이 아니더라도 문자열로 강제 타입 변환되어 삽입된다.

4. 불리언 타입 (boolean)

  • 논리적 참, 거짓을 나타내는 true, false.
  • 조건문에서 자주 사용.

5. undefined 타입

  • 자바스크립트 엔진이 변수를 초기화 할 때 사용하는 값.
  • 변수를 참조했을 때 undefined가 반환된다면 참조한 변수가 선언 이후 값이 할당된 적이 없는 변수라는 것을 알 수 있다.
  • 값이 없다는 것을 명시하고 싶은 경우엔 undefined 가 아니라 null을 할당한다.

6. null 타입

  • 변수에 값이 없다는 것을 의도적으로 명시(Intentional absence) 할 때 사용.
  • 변수에 null을 할당하는 것은 변수가 이전에 참조하던 값을 더 이상 참조하지 않겠다는 의미.
  • 이전에 할당되어 있던 값에 대한 참조를 명시적으로 제거하는 것을 의미.
  • 함수가 유효한 값을 반환할 수 없는 경우, 명시적으로 null을 반환하기도 함.

7. symbol 타입

  • ES6에서 새롭게 추가된 7번째 타입으로 변경 불가능한 원시 타입의 값.
  • 주로 이름의 충돌 위험이 없는 객체의 유일한 프로퍼티 키(property key)를 만들기 위해 사용.
  • Symbol 함수를 호출해 생성. 심볼 값은 노출되지 않으며 다른 값과 절대 중복되지 않는 유일무이한 값.

2. 데이터 타입의 필요성

1. 데이터 타입에 의한 메모리 공간의 확보와 참조

  • 변수를 선언하고 숫자 값을 할당하면, 할당 값의 데이터 타입에 따라 확보해야 할 메모리 공간의 크기가 결정된다.
  • 값을 참조하려면 한번에 읽어 들여야 할 메모리공간의 크기를 알아야 한다. 이 때 자바스크립트 엔진이 변수 값의 타입을 인식하여 타입의 크기만큼 읽어들인다.

2. 데이터 타입에 의한 값의 해석

  • 모든 값은 메모리에 2진수로 저장된다. 메모리에 저장된 값은 데이터 타입에 따라 다르게 해석 될 수 있다.
  • 변수에 할당된 값의 타입에 따라서 메모리 공간의 주소에서 읽어 들인 2진수를 데이터 타입에 맞게 값을 해석한다.

3. 동적 타이핑 (Dynamic typing)

1. 동적 타입 언어와 정적 타입 언어

  • 자바스크립트는 정적 타입 언어와는 다르게 변수를 선언할 때 타입을 선언하지 않는다.
  • 어떠한 데이터 타입의 값이라도 자유롭게 할당할 수 있다.
  • typeof 연산자는 자신의 뒤에 위치한 피연산자의 데이터 타입을 문자열로 반환.
  • 값을 할당하는 시점에 변수의 타입이 동적으로 결정되고, 변수의 타입을 언제든지 자유롭게 변경할 수 있다.
  • 자바스크립트 변수는 선언이 아닌 할당에 의해 타입이 결정.
  • 재할당에 의해 변수의 타입은 언제든지 동적으로 변할 수 있다.
  • 변수는 타입을 갖지 않는다. 하지만 값은 타입을 갖는다.

2. 동적 타입 언어와 변수

  • 동적 타입 언어의 변수는 값을 확인하기 전에는 타입을 확신할 수 없다.
  • 자바스크립트 엔진에 의해 암묵적으로 타입이 자동 변환 되기도 한다.
  • 유연성은 높지만 신뢰성은 떨어진다.

3. 변수를 사용할 때 주의할 사항

  • 변수는 꼭 필요한 경우에 한해 제한적으로 사용한다. 변수의 개수가 많으면 많을수록 오류가 발생할 확률은 높아진다.
  • 변수의 유효 범위(스코프)는 최대한 좁게 만들어 변수의 부작용을 억제해야 한다.
  • 전역 변수는 최대한 사용하지 않도록 한다.
  • 변수보다는 상수를 사용해 값의 변경을 억제한다.
  • 변수 이름은 변수의 목적이나 의미를 파악할 수 있도록 네이밍한다.

javascript - expression & statement

Posted on 2020-02-20 In Learn web development , javascript

1. 표현식과 문

(값, 리터럴, 표현식, 문, 세미콜론, 표현식인 문과 표현식이 아닌 문.)

1. 값 (value)

  • 값은 식(표현식, expression)이 평가(evaluate)되어 생성된 결과.
  • var sum = 10 + 20; 에서 10 + 20 은 식, 숫자 값 30이 생성되어 값으로 저장된다.

2. 리터럴 (literal)

  • 사람이 이해할 수 있는 문자, 약속된 기호를 사용하여 값을 생성하는 표기방식.
  • 문자(아라비아 숫자, 알파벳, 한글 등) 또는 미리 약속된 기호로 표기한 코드.
  • 값을 생성하기 위해 미리 약속한 표기법(notation)
  • 숫자 리터럴(정수, 부동소숫점, 2진수, 8진수, 16진수), 문자열 리터럴, 불리언 리터럴, null 리터럴, undefined 리터럴, 객체 리터럴, 배열 리터럴, 함수 리터럴, 정규표현식 리터럴.

3. 표현식 (expression)

  • 값으로 평가될 수 있는 문.
  • 표현식이 평가되면 값이 생성되거나, 값을 참조.
  • 리터럴, 식별자(변수, 함수이름), 연산자, 함수 호출의 조합.
  • 리터럴도 값으로 평가. 리터럴도 표현식.
  • 변수 식별자를 참조하면 변수 값으로 평가. 값을 생성하지는 않지만 값으로 평가되므로 표현식.
  • 표현식과 표현식이 평가된 값은 동등한 관계. 동치(equivalent). 1 + 2 = 3, 1 + 2 와 3 은 동치.

4. 문 (statement)

  • 프로그램을 구성하는 기본 단위이자 최소 실행 단위.
  • 문의 집합 -> 프로그램.
  • 문을 작성하고 순서에 맞게 나열하는 것 -> 프로그래밍.
  • 여러 토큰들로 구성됨. 토큰(token) : 문법적으로 더 이상 나눌 수 없는 코드의 기본 요소. 키워드, 식별자, 연산자, 리터럴, 세미콜론(;), 마침표(.) 등.
  • 명령문. 문은 컴퓨터에 내리는 명령.
  • 선언문, 할당문, 조건문, 반복문.

5. 세미콜론과 세미콜론 자동 삽입 기능

  • 세미콜론(;)은 문의 종료를 나타냄.
  • 단, 0개 이상의 문을 중괄호로 묶은 코드 블록 {…} 뒤에는 세미콜론을 붙이지 않는다. 이들 코드 블록은 자체 종결성(self closing)을 갖기때문.
  • 세미콜론은 생략 가능. 자바스크립트 엔진이 문의 끝이라고 예측되는 지점에 세미콜론 자동 삽입 기능 (ASI, automatic semicolon insertion) 이 암묵적으로 수행됨.

6. 표현식인 문과 표현식이 아닌 문

  • 변수 선언문은 값으로 평가될 수 없으므로 표현식이 아닌 문이다. 값처럼 사용할 수 없다.
  • 1, 2, 1 + 2, x = 1 + 2는 표현식이다. 값처럼 사용할 수 있다.
  • x = 1 + 2; 는 표현식이면서 완전한 문이기도 하다.
  • 구별방법 : 변수에 할당해 보는 것. 표현식인 문은 값으로 평가되므로 변수에 할당할 수 있다. 하지만 표현식이 아닌 문은 값으로 평가할 수 없으므로 에러가 발생한다.

* 완료 값 (Completion value)

  • 표현식이 아닌 문은 완료 값 undefined를 반환.
  • 표현식인 문은 언제나 평가된 값을 출력.
